**Second attempt (session):** Correctly identified the structure flaw and moved to recursive node-by-node comparison.

function isSameTree(p, q) {
    if(p.val !== q.val) return false

**Final correct solution:**

```js
function isSameTree(p, q) {
    if(p === null && q === null) return true;
    if(p === null || q === null) return false;
    if(p.val !== q.val) return false;
    const isLeftSame = isSameTree(p.left, q.left);
    const isRightSame = isSameTree(p.right, q.right);
    return isLeftSame && isRightSame;
}
```

## Where He Got Stuck

Two places:
1. **Early return bug** — wrote `if(p.val === q.val) return true` which short-circuited before checking subtrees
2. **Null base case** — understood it needed null handling but struggled to connect what returning `true` for `null, null` meant

Unlocked by tracing through a concrete example step by step.

## Mistakes Made

1. **First approach missed structure** — flattening to arrays loses position information. `[1,2,3]` can come from different tree shapes.
2. **Short-circuit bug** — matching values at the current node is not enough to return true. Must check subtrees too.
3. **Missing base case** — no null guard before accessing `.val` → crash on leaf's children.
4. **Struggled to articulate what `null, null → true` means** — took a concrete trace to land.

## Key Insight

Check the current nodes first (null cases, then value). If everything checks out at this node, delegate to recursion for both subtrees. Only return true when every position in both trees agrees.
