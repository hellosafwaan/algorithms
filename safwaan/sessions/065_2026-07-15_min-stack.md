# Session: Min Stack — 2026-07-15

## What He Attempted
Phase 4 curriculum #2 — the first genuinely new stack shape today (no fundamentals-module pattern maps onto this one). **Disclosed upfront that he'd watched a NeetCode video.** Brought a class-based `MinStack` implementation using two parallel stacks (`stack` for values, `minStack` tracking the running minimum at each depth).

## Where He Got Stuck
`pop()` had a real bug:
```js
MinStack.prototype.pop = function() {
    this.stack.pop(value);
    this.minStack.pop(value);
};
```
`value` was never a parameter of `pop` — it only existed as a parameter on `push`. Referencing it here would throw a ReferenceError at runtime.

## Mistakes Made
The `value` reference bug above. Caught in one guided question: "what does `value` refer to inside `pop`?" — he immediately identified it should just be `pop()` with no arguments (`Array.prototype.pop()` takes none anyway).

## Key Insight
The design: `stack` holds the actual pushed values; `minStack` holds, at each index, the minimum of everything in `stack` up to that point. On every push, the new `minStack` entry is `Math.min(newValue, currentMinStackTop)` (or just the value if `minStack` is empty). This means `minStack`'s top is always the current overall minimum — `getMin()` is a straight O(1) peek, no scanning.

After the fix, submitted and accepted (41.11th percentile runtime, 42.18th percentile memory).

**Extension discussion (not implemented):** raised whether `minStack` needs a new entry on *every* push, even when the pushed value isn't a new minimum. He engaged fully and correctly self-derived, through pure questioning (no answers given): if the value about to be popped off `stack` equals the current top of `minStack`, that means it was the current minimum, so `minStack` needs to pop too — otherwise `minStack` should stay untouched. This is the core mechanism of the space-optimized lazy-min-stack variant. He explicitly asked to defer *implementing* it to a future session ("note this something to look at later").

## Complexity Reached
Time: O(1) for push, pop, top, getMin — all direct array operations.
Space: O(n) for both stacks combined (current, unoptimized version pushes to `minStack` on every single push).
