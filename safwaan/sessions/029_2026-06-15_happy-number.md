## Key Insight

**Cycle detection = repeated value.** If you're in an infinite loop, you must eventually revisit a value. Track seen values in a Set; return false on a repeat. This is the general technique for detecting cycles when you have a transformation function.

**Digit extraction without strings:** `n % 10` extracts the last digit; `Math.floor(n / 10)` removes it. Loop while `n > 0`. This is pure arithmetic — faster than `String()` conversion and avoids string allocation overhead.

## Complexity Reached

- **Time: O(log n)** — the first iteration processes all digits of n (log₁₀(n) digits). After one or two steps, any number collapses to ≤ 243 (max for 3 digits: 3 × 81). All subsequent iterations are O(1) with at most a constant number of steps.
- **Space: O(1)** — the Set is bounded by ~243 entries regardless of n. Cycle is always entered below 243.
